# EyeScroll - 眼球追踪滚动应用设计规范

## 1. 项目概述

### 1.1 目标
开发一个 macOS 菜单栏应用，通过 MacBook 内置摄像头追踪用户眼球运动，实现"向下看即滚动"的免手阅读体验。

### 1.2 核心功能
- 使用内置摄像头进行实时眼球追踪
- 当用户视线移向屏幕下方区域并停留一定时间后，自动触发向下滚动
- 适用于所有 macOS 应用（浏览器、PDF 阅读器、文本编辑器等）

### 1.3 技术选型
- **语言**: Python 3.11+
- **眼球追踪**: MediaPipe Face Mesh / Iris
- **摄像头捕获**: OpenCV
- **系统滚动控制**: PyObjC (macOS Accessibility API)
- **菜单栏 UI**: rumps

## 2. 交互设计

### 2.1 视线区域划分

```
┌─────────────────────────────┐
│                             │  ← 上方静默区 (0-60%)
│                             │     正常阅读，不触发任何动作
├─────────────────────────────┤
│                             │
│        阅读区域              │  ← 中央阅读区 (60-75%)
│                             │     主要活动区域
├─────────────────────────────┤
│     ▼ 滚动触发区 ▼           │  ← 下方滚动区 (75-100%)
│   （停留 0.5s 后触发滚动）     │     停留触发，防止误触
└─────────────────────────────┘
```

### 2.2 状态机设计

```
         ┌───────────────────┐
         │                   │
         ▼                   │
    ┌─────────┐    看向下方    │
    │  IDLE   │──────────────┐│
    │ (阅读中) │   75%区域    ││
    └─────────┘              ││
         ▲                   ▼│
         │            ┌────────────┐
    看向阅读区        │  DWELLING  │
    或上方           │  (停留计时)  │
         │            └────────────┘
         │                   │
         │            停留 > 0.5s
         │                   │
         │                   ▼
         │            ┌────────────┐
         └────────────│  SCROLLING │
           视线离开    │  (持续滚动)  │
           滚动区      └────────────┘
```

### 2.3 核心参数

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| `SCROLL_ZONE_RATIO` | 0.25 | 0.1-0.4 | 屏幕下方区域占比 |
| `DWELL_TIME_MS` | 500 | 200-1000 | 停留触发时间（毫秒）|
| `SCROLL_DISTANCE` | 80 | 20-200 | 每次滚动像素数 |
| `SCROLL_INTERVAL_MS` | 100 | 50-500 | 滚动事件间隔 |
| `DETECTION_CONFIDENCE` | 0.5 | 0.3-0.9 | 人脸检测置信度阈值 |

## 4. 数据流

### 4.1 主数据流

```
摄像头帧 (30fps)
    │
    ▼
┌─────────────────┐
│ MediaPipe       │
│ Face Mesh       │
│ - 检测人脸      │
│ - 提取虹膜坐标  │
└────────┬────────┘         │
         ▼
┌─────────────────┐
│ 坐标归一化      │
│ (x, y) → [0,1]  │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 区域判定        │
│ y > 0.75?       │
│ 是 → DWELLING   │
│ 否 → IDLE       │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 状态机处理      │
│ - 计时器管理    │
│ - 状态转换      │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 滚动执行        │
│ CGEventScroll   │
│ (当前鼠标位置)  │
└─────────────────┘
```

### 4.2 MediaPipe 虹膜检测

MediaPipe Face Mesh 提供 468 个面部关键点，其中虹膜相关点为：
- 左眼虹膜中心: 索引 468
- 右眼虹膜中心: 索引 473

```python
# 伪代码
landmarks = face_mesh.process(frame)
left_iris = landmarks[468]  # (x, y, z) 归一化坐标
right_iris = landmarks[473]

# 取两眼中心作为视线位置
gaze_x = (left_iris.x + right_iris.x) / 2
gaze_y = (left_iris.y + right_iris.y) / 2
```

## 5. 异常处理

### 5.1 异常场景处理

| 场景 | 检测方式 | 处理方式 |
|------|----------|----------|
| **未检测到人脸** | MediaPipe 返回空结果 | 显示"请面向摄像头"提示，暂停滚动 |
| **检测到多人** | 检测到多个人脸 | 选择面积最大的人脸 |
| **光照不足** | 图像平均亮度低 | 显示"光线不足"警告，降低置信度阈值 |
| **用户眨眼** | 连续帧虹膜丢失 < 100ms | 保持上一个有效状态 |
| **摄像头被占用** | OpenCV 打开失败 | 启动时检测，提示关闭其他应用 |
| **权限未授权** | 系统权限检查 | 引导用户到系统设置授权 |

### 5.2 权限要求

应用需要以下 macOS 权限：
1. **摄像头权限** - 用于眼球追踪
2. **辅助功能权限** - 用于发送滚动事件
